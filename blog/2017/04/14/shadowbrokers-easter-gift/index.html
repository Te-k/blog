ShadowBrokers Easter Gift

Apr 14, 2017 · 8 minute read
malware

UPDATE IN PROGRESS

So the Shadow Brokers finally published some Windows tools, apparently from the NSA toolkit. There are 3 different directories :

  • ODDJOB : dropper which download and launch an encrypted payload (with builder and instruction to install the C2 server)
  • SWIFT : information on operations targeting SWIFT networks
  • Windows attack tools

Thanks to @x0rz you can find all these files on github. I have summarized here the information I have seen on twitter, and some notes from my tests. You should definitely follow @hackerfantastic, @x0rz, Kevin Beaumont, nex on Twitter to get more update.

April 15th Update: So Microsoft published a statement saying that all the vulnerabilities are already fixed in recent Windows version, most of them with MS17-010 in March. So it seems that everyone (including me) was a bit quick with conclusions yesterday. It is still surprising to see 3 major vulnerabilities related to ShadowBrokers fixed at the same time in March, which may mean that NSA actually disclosed these vulnerabilities to Microsoft (there is no credits in the Bulletin). This annoucement reduce the risk of this publication but there are still a lot of interesting information in this publication, attacks on SWIFT being one of them, but also better knowledge on tools and implants used by NSA (and many of them are still not analyzed fully). It can have an important impact on NSA operations by “burning” these tools, but it can also allow to link NSA operations to more attacks (published or yet to be published) and get a better insight on their strategies and objectives.

Oddjob

ODDJOB seems to be an dropper communicating through HTTP to download an encrypted payload and launch it on the system. Each implant has a specific id generated by the builder which is used to decrypt the payload. A builder is provided to create both implant and payload (another tool called FELONYCROWBAR is also mentioned for this purpose but not provided):

The .hta files really use the command line builder which has the following options:

USAGE============================================
oddjob_config -configure <oddjob.exe> <ARGS>
oddjob_config -dumpconfig <oddjob.exe>
oddjob_config -decrypt <surveystring>
oddjob_config -encryptpayload <oddjob.exe> <payload.exe> <encryptedpayload.exe>
<beaconstring> <type, 1=EXE, 2=DLL>
oddjob_config -decryptpayload <oddjob.exe> <encryptedpayload.exe> NOTE: replaces
 encrypted file
oddjob_config -genpubkey
oddjob_config -genprivatekey

SWIFT

The SWIFT directory seems to contains information on operations against SWIFT networks. Most of the documents seems to concern a successful attack on the Eastnets network.

Partial list of files :

  • DSquery Belgium DC.xlsx : List of servers on the internal EastNets Belgium Bureau obtained with dsquery (1911 servers listed with their name, DNS name, operating system and their created datem changed date and last logon)
  • dsquery_Query_computers_from_MAIL001.txt : Same list of servers, likely in another EastNets bureau obtained through AD query with dsquery
  • DSquey Dubai enDCBACKUP.xlsx : list of servers in the EastNets Dubai bureau (still obtained with dsquery, 1911 servers)

Exploits

Exploit directory, every exploits comes with an xml describing options and targets:

  • EASYBEE : exploit for MDaemon WorldClient (Web interface for MDaemon email server) and MDaemon WebAdmin (remote administration tool for MDaemon servers)
  • EASYPI : exploit for Lotus CC:Mail for windows NT/2000/XP/2003
  • ECLIPSEDWING : exploit for MS08-67 (remote code execution in Windows RPC) for Windows 2000/XP/2003
  • EDUCATEDSCHOLAR : exploit for MS09-050 (remote code execution in SMBv2) on Windows Vista and 2008.
  • EMERALDTHREAD : remote code execution through SMB or NBT on Windows XP / 2003. Fixed by MS10-061
  • EMPHASISMINE : exploit in Lotus Domino through IMAP, maybe 0day (xml file)
  • ENGLISHMANSDENTIST : likely local code execution on Outlook by email using a vulnerability in the TNEF format. It is interesting to see that the exploit is language dependent and that only English/German/Korea and Chinese are proposed. Microsoft said in a statement that as supported plarforms are not vulnerablem they won’t fix this vulnerability.
  • ERRATICGOPHER : remote code execution through RPC in XP and 2003, fixed before Vista release. (xml)
  • ESKIMOROLL : Kerberos exploit targeting 2000/2003/2008 servers, MS14-068 according to @gentilkiwi and Microsoft, binary timestamp is in 2011…
  • ESTEEMAUDIT : remote code execution in RDP targeting XP and 2003, 0day that won’t be fixed by Microsoft because it only affect outdated platforms
  • ETERNALROMANCE : remote code execution on Windows XP/2003/Vista/2008 using SMB fixed by MS17-010
  • ETERNALSYNERGY : remote code execution in SMB targeting Windows 8 and 2012, fixed by MS17-010
  • EWOKFRENZY : remote code execution pre-auth on Lotus Domino version 6.5.4 and 7.0.2 through IMAP, potential 0day (it seems to be old versions, last one is 9.0.1)
  • EXPLODINGCAN : remote code execution on IIS6 on Windows 2003 through PROPFIND bug, CVE-2017-7269. Exploited in the wild in July/August 2016. Public exploit already available, thanks @GossiTheDog. As only plaforms not supported anymore by Microsoft are vulnerable, it won’t be fixed
  • ZIPPYBEER : only python exploit, it seems to be an authenticated remote code execution in SMB, no information on the versions vulnerable, potential 0day

Specials

Dunno what is special there, but lot of exploits involved:

  • ETERNALBLUE : SMB remote code execution on Windows XP/2003/Vista/2008/7, confirmed 0 day fixed by MS17-010. Here is a nice video showing the exploit by @hackerfantastic/. Conformed working on Win7x86.
  • ETERNALCHAMPION : another SMB remote code execution on Windows XP/2003/Vista/2008/7, confirmed 0day fixed by CVE-2017-0146 & CVE-2017-0147

Touches

This folder seems to gather utilities :

  • ARCHITOUCH : seems to be a test of SMB connection to get
  • DOMAINTOUCH : something related to SMB
  • ECLIPSEDWINGTOUCH : Scanner a server for MS08-076 SMB vulnerability
  • EDUCATEDSCHOLARTOUCH : Scanner for a EDUCATEDSCHOLAR SMB vulnerability MS09-050
  • EMERALDTHREADTOUCH : Scanner for MS10-061
  • ERRATICGOPHERTOUCH : Scanner for ERRATICGOPHERTOUCH vulnerability
  • ESTEEMAUDITTOUCH : Scanner for remote code execution in RDP targeting XP and 2003, 0day that won’t be fixed by Microsoft
  • EXPLODINGCANTOUCH : Scanner for CVE-2017-7269
  • IisTouch : Scanner for IIS, get version, OS and WebDav information
  • Namedpiptouch : look for specific SMB named pipes

Resources

The Ops directory seems to include interesting resources for operations :

  • Databases/DriverList.db includes a list of 22000+ legitimate Windows driver and hashes (you can find this list here)
  • Databases/mac.db includes lists of IAB and OUOI (used to dermine manufacturer of a network card based on its MAC address)
  • Databases/SimpleProcesses.db : list of process names for different products (full list here. It also includes process names used by different NSA products:
  • Uploads contains promiscdetect, a tool to detect if your network interface is in promiscuous mode (promiscdetect_safe.exe is the same tool without copyright infos according to @Hexacorn). I have not information yet on rar.exe and yak_min_install.exe

  • Tools :

    • boxwine seems to be a kind of encryption/encoding algorithm for username and password
    • dotrenamer.pl rename files with a dot in filename (??)
    • fb_logs.py manipulate log files from different tools

  • PyScripts contains python tools:

    • lib contains python libraries for other tools
    • ripper is a credential dumper for Firefox/Chrome/Skype

lib

This directory includes dlls which are used by many other tools:

  • coli-0.dll : coli stands for COmmand LIne
  • tibe-2.dll : SMB networking wrapper (used by EternalRomance)
